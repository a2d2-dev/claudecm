**User story.**
As a claudecm developer, I want a pinned `testdata/codex/{happy,edge}/` corpus that exercises every NFR-T1 case so the Codex adapter cannot regress silently.

**Acceptance criteria.**
- `testdata/codex/happy/canonical_minimal/{config.toml,auth.json}` + `canonical_maximal/...` exist.

- Each fixture has golden expected outputs for `Import`, `Plan`, `Apply`, `Project`.
- Round-trip `import → switch → export → explain` byte-identical on owned-key scope for every fixture.

## Implementation Notes / Decisions (post-review followup)

The fixture matrix landed in two passes; the review followup pass
reconciled the AC list against the shipped corpus and formalized
what stayed deferred. Same shape as the E3-S7 reconciliation.

Shipped case names under `testdata/codex/`:

- `happy/minimal/`, `happy/kitchen-sink/`, `happy/openai-provider/`,
  `happy/anthropic-provider/` — canonical minimal + maximal
  round-trip goldens (Import, Plan, Apply, Project).
- `edge/auth-only/`, `edge/config-only/` — partial-file coverage
  (each file present alone, the other absent).
- `edge/unknown-keys-preserved/` — merge-preserve round-trips
  non-allowlisted keys byte-verbatim.
- `edge/missing/`, `edge/empty-files/`, `edge/whitespace-only/` —
  early-exit `ErrNoConfig` fan-in.
- `edge/malformed-config/`, `edge/malformed-auth/` —
  `ErrParseFailed` per file.
- `edge/symlink-out-of-home/` — config.toml resolves outside HOME,
  asserts `errors.Is(err, ErrOutsideHome)` on the read side.
- `edge/symlink-out-of-home-auth/` — symmetric to the config case
  but targeting auth.json; ensures out-of-HOME containment applies
  to both files (added in the review followup).
- Plans-projection golden `expected/plans.json` — shallow
  `{Tool, Target, OwnedKeys, Reason}` snapshot per `WritePlan` for
  every case that carries a `profile.yaml` (added in the review
  followup; closes the AC "Plan golden" gap).

Deferred AC directories and rationale:

- `edge/bom/`, `edge/crlf/`, `edge/comments/`, `edge/mixed_indent/`
  — TOML-parser-layer concerns. The codex adapter delegates all
  four to `internal/codex/toml`'s doc-model wrapper, which already
  has direct unit coverage in `codex/toml/doc_test.go`. A fixture
  golden here would either shadow the parser test (no new signal)
  or require adapter-level policy decisions that do not exist in
  v1 (e.g. "should Import normalize CRLF to LF?"). Land the
  policy first, then a fixture.
- `edge/separate_fs/` — cross-filesystem rename fallback. Out of
  `~/.codex/` scope for v1; the write-path treats cross-fs as an
  error rather than a fallback per NFR-S1. Would need a mount
  fixture that CI does not provide.

Both clusters are tracked in the follow-up stub
`docs/plan/stories/E4-FOLLOWUP-codex-edge-fixtures.md`.

Two tripwires guard against future silent gaps (same shape as
E3-S7):

- `TestFixtureGoldensAreValidJSON` — every `expected/*.json`
  round-trips through `json.Unmarshal`.
- `compareCount` inside `runFixtureCase` — a non-error-only case
  that skipped every golden compare fails rather than passing.
- `readIfExists` now takes `*testing.T` and `t.Fatalf`s on any
  non-ENOENT error so a permissions bug in the harness cannot
  masquerade as an auth-elision skip.
