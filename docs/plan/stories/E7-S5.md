# E7-S5 — `PartialFailure` structured error + per-file `committed | rolled-back | untouched`

**User story.**
As a claudecm user, I want a failed `switch` (or `restore`) across multiple tool files to print a clear, structured per-file status (`committed`, `rolled-back`, `untouched`) with backup paths so I know exactly where I stand.

**Acceptance criteria.**
- `PartialFailure` carries `PerFile []FileStatus` where each entry has `Path`, `Status` (`committed|rolled-back|rollback-failed|untouched`), `BackupPath`, and optional `Err`.
- The CLI formatter prints a human-friendly table; `--output json` emits the same in JSON.
- Exit code on `PartialFailure`: 2 if all rolled back cleanly (recoverable), 3 if any rollback failed (manual intervention required).
- The error surfaces in `switch`, `restore`, and any future multi-file command.

**PRD/architecture refs.** PRD FR-16; Architecture §5; coding-standards "Error Wrapping" exit codes.

**Test plan.**
- Happy: clean two-phase failure → exit 2, both files reported `rolled-back` with backup paths.
- Edge: rollback-failed on one file → exit 3, both errors visible, JSON output validates.
