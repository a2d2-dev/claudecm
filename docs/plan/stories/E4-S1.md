# E4-S1 — `codex`: `Detect` + `Files` + per-file owned-key allowlist `var`s

**User story.**
As a claudecm developer, I want the Codex adapter to own exactly two files (`~/.codex/config.toml`, `~/.codex/auth.json`), publish per-file owned-key allowlists as exported Go `var`s, and detect Codex installs from those file paths.

**Acceptance criteria.**
- `codex.Detect()` returns installed when either file exists; missing both → not installed.
- `codex.Files()` returns two `OwnedFile` entries with correct formats (`TOML`, `JSON`).
- `codex.OwnedKeysConfigToml` is an exported `var []KeyPath` equal to (verbatim):
  - `model`
  - `model_provider`
  - `model_providers.<name>.base_url`
  - `model_providers.<name>.wire_api`
  - `model_providers.<name>.env_key`
  - `model_providers.<name>.name`
- `codex.OwnedKeysAuthJSON` is an exported `var []KeyPath` containing `OPENAI_API_KEY` plus the frozen Codex auth fields per PRD §4.7.
- `<name>` is a glob path — the adapter handles arbitrary provider names declared in `model_providers`.
- Adding a key requires a paired PRD §4.7 edit (coding-standards rule 4).

**PRD/architecture refs.** PRD §4.7 (Codex); Architecture §3.1; coding-standards rule 4.

**Test plan.**
- Happy: both files present → detect installed; allowlists round-trip through `Files()`.
- Edge: only `auth.json` present → still installed; `config.toml` with provider `relay-a` → owned keys glob-resolved.

**No fallback writes.** N/A — declarations only.

**Complexity.** S.

**Dependencies.** E3-S1.
