# E6-S1 — `cmd/current`

**User story.**
As a claudecm user, I want `claudecm current` to show the active profile and per-tool effective summary so I can confirm what's loaded before running an expensive coding session.

**Acceptance criteria.**
- Given an active profile, when `current` runs, then it prints the active profile name and the minimum effective values per tool (base_url, model, key-source label).
- Default redaction applies (NFR-S8); `--reveal` opts in.
- External drift (E5-S4) prints a one-line warning when present.
- `--output json` emits machine-readable form.
- Given no active profile, when `current` runs, then it prints `no active profile` and exits 0 (it's an informational command, not an error).

**PRD/architecture refs.** PRD FR-6, NFR-S8; Architecture §6, §7.

**Test plan.**
- Happy: profile active → summary shown.
- Edge: no active profile → "no active profile" message; drift detected → warning printed; `--reveal` shows full key + stderr notice.

**No fallback writes.** Read-only.

**Complexity.** S.

**Dependencies.** E5-S5.
