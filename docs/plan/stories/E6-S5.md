# E6-S5 — `cmd/edit`: `$EDITOR` default + `--set key=value` + `--dry-run`

**User story.**
As a claudecm user, I want `claudecm edit <name>` to launch `$EDITOR` (fallback `vi`) on a temp copy of the profile YAML and validate on save, with a `--set key=value` flag for scripted edits.

**Acceptance criteria.**
- Given `edit <name>` interactively, when invoked, then a temp copy of the profile YAML is written to a temp dir, `$EDITOR` is launched, and on save claudecm re-parses + re-validates.
- Given invalid YAML or schema on save, when re-parse fails, then the edit is rejected, original profile preserved, and a clear error printed.
- Given `--set key=value` (repeatable), when invoked, then values are applied non-interactively with the same validation.
- Given `--dry-run`, when invoked, then the unified diff is printed and no write occurs.
- If the edited profile is the active profile and writes a value that affects tool-config rendering, the command does NOT auto-apply; user must `switch` to activate.

**PRD/architecture refs.** PRD FR-2, FR-15, NFR-M1; Architecture §7.

**Test plan.**
- Happy: `edit` → save valid YAML → profile updated.
- Edge: corrupt YAML on save → rejected, original preserved; `--set tools.codex.model=foo --dry-run` → unified diff printed, no write; `$EDITOR` unset → falls back to `vi`.

**No fallback writes.** Bad YAML on save rejects; original preserved. NFR-S1 / NFR-M1.

**Complexity.** M.

**Dependencies.** E1-S1, E1-S7.
