# E6-S2 — `cmd/switch` refactor: direct-write via two-phase commit, pre-apply diff, `--dry-run`, `--yes`

**User story.**
As a claudecm user, I want `claudecm switch <name>` to atomically write the new profile's intent to every owned tool file using the two-phase commit, show me an explain-style pre-apply diff first, and refuse to touch non-owned keys without my explicit `--yes`.

**Acceptance criteria.**
- Given a profile name argument, when `switch` runs, then it computes per-tool `WritePlan`s via the adapters' `Plan`, prints the pre-apply diff (owned-key old → new, non-owned change list with "preserved" / "may shift" annotations), and routes writes through `internal/commit.Commit`.
- Given the diff touches keys claudecm does NOT own (impossible for owned scope by definition, but `--strict` widens the check), when in interactive mode, then it prompts `y/N`; when non-interactive without `--yes`, then it aborts.
- Given `--dry-run`, when `switch` runs, then it prints the diff and exits 0 with no lock, no backup, no write.
- Given a successful switch, when `switch` returns, then it reports the new active profile plus every backup path created.
- The newly active profile must be observable via `current` immediately; SM-1 (end-to-end < 2 s) is enforced in CI smoke.
- Direct sequencing of `writepath.Apply` across files is **forbidden** (coding-standards rule 13); switch must use `commit`.

**PRD/architecture refs.** PRD FR-4, FR-5, FR-15, FR-16, NFR-S6, NFR-S7; Architecture §5, §7.

**Test plan.**
- Happy: switch from `official` to `relay-a` → diff printed, both files updated, backups present, `current` confirms.
- Edge: non-interactive without `--yes` → abort; `--dry-run` → no files mutated; concurrent edit between read and write → exit code 2 with backup retained.

**No fallback writes.** Every write routes through `writepath.Apply` via `commit`. Refuse-on-malformed parses propagate to a clean user-facing error. NFR-S1.

**Complexity.** L.

**Dependencies.** E5-S5, E7-S1..E7-S5, E3-S5, E4-S5.
