# E3-S2 — `claudecode`: `Detect` + `Files` + owned-key allowlist as Go `var`

**User story.**
As a claudecm developer, I want the Claude Code adapter to declare the user-scope `~/.claude/settings.json` as the **only** owned file and publish the frozen owned-key allowlist as a single exported Go `var`, so merge-preserve has one source of truth and CI can guard it.

**Acceptance criteria.**
- `claudecode.Detect()` returns `Presence{Installed: ...}` based on the existence of `~/.claude/settings.json` (project-scope files are never read).
- `claudecode.Files()` returns exactly one `OwnedFile` at `~/.claude/settings.json`, format JSONC.
- `claudecode.OwnedKeysSettingsJSON` is an exported `var []KeyPath` equal to (verbatim):
  - `env.ANTHROPIC_API_KEY`
  - `env.ANTHROPIC_BASE_URL`
  - `env.ANTHROPIC_AUTH_TOKEN`
  - `env.ANTHROPIC_MODEL`
  - `env.ANTHROPIC_SMALL_FAST_MODEL`
  - `env.CLAUDE_CODE_USE_BEDROCK`
  - `env.CLAUDE_CODE_USE_VERTEX`
- Project-scope `.claude/settings.json` and `.claude/settings.local.json` are never referenced in code (CI grep guard).
- Adding any key to this `var` requires a paired PRD §4.7 edit (coding-standards rule 4).

**PRD/architecture refs.** PRD §4.7 (Claude Code), §5 (project scope OUT); Architecture §3.1; coding-standards rule 4.

**Test plan.**
- Happy: with `~/.claude/settings.json` present → `Detect` returns installed; `Files()` returns the single owned file.
- Edge: project-scope file present at `./.claude/settings.json` → ignored by `Detect` and not in `Files()`.

**No fallback writes.** N/A — read-only declaration.

**Dependencies.** E1-S2, E3-S1.
