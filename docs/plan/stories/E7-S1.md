# E7-S1 — `internal/commit`: `Commit` interface + `StagedTxn` types

**User story.**
As a claudecm developer, I want the two-phase-commit interface declared up front so adapters and commands compile against a stable contract before the ordered-rename body lands.

**Acceptance criteria.**
- `internal/commit/commit.go` declares `Commit` interface (`Stage(plans []WritePlan) (StagedTxn, error)` and `Commit(StagedTxn) (CommitReport, error)`), `StagedTxn`, `CommitReport`, `PartialFailure`.
- `CommitReport.PerFile` enumerates `committed | rolled-back | untouched` per target.
- `PartialFailure` is a structured error type implementing `error`, returned when phase 2 partially fails.
- Direct sequencing of `writepath.Apply` calls across files is **forbidden** (coding-standards rule 13); commands must use this package.

**PRD/architecture refs.** PRD FR-16; Architecture §5; coding-standards rule 13.

**Test plan.**
- Happy: types compile, error type matches Go's error contract.
- Edge: zero-plan input → `Stage` returns an empty transaction (no-op) and `Commit` is a no-op.

**No fallback writes.** N/A — interface only.

**Complexity.** S.

**Dependencies.** E2-S1.
