# E6-S7 — `cmd/restore`: `--list`, `--latest`, `--id`, `--dry-run`, `--yes`

**User story.**
As a claudecm user, I want `claudecm restore` to list backups and revert any owned file to a chosen backup, with the same write-path safety (FR-5) as `switch`.

**Acceptance criteria.**
- Given `restore --tool <claude-code|codex> --list`, when run, then it enumerates every backup for that tool's owned files, newest first, with timestamp + size + provenance.
- Given `--latest`, when run, then it restores the most recent backup of every owned file for that tool using the FR-5 pipeline (new backup of current file taken first).
- Given `--id <backup-id>`, when run, then it restores that specific backup.
- Given `--dry-run`, when run, then the explain-style pre-apply diff is printed and no write occurs.
- `restore` never deletes a backup; retention is governed by NFR-R1.
- A restored switch is observable via `current` immediately.

**PRD/architecture refs.** PRD FR-12, FR-5, FR-15, NFR-R1, NFR-R3; Architecture §4, §7.

**Test plan.**
- Happy: switch → restore --latest → file content matches pre-switch byte-for-byte.
- Edge: `--id` for a deleted backup id → error; `--dry-run` → diff only; restore touches both files for Codex → routes through `commit` (E7).

**No fallback writes.** Routed through `writepath.Apply` (and `commit` for multi-file). NFR-S1.

**Complexity.** M.

**Dependencies.** E2-S5, E7-S5, E3-S5, E4-S5.
