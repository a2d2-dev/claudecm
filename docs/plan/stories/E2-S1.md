# E2-S1 — `WritePlan`, `ApplyReport`, `OwnedFile`, `KeyPath` types

**User story.**
As a claudecm developer, I want the value types that flow into and out of `writepath.Apply` defined first, so adapters and the commit package can compile against a stable contract before the pipeline body lands.

**Acceptance criteria.**
- `WritePlan` carries: target file path, format (JSON/JSONC/TOML), owned-key allowlist, current bytes, intended bytes, owned-key diff, tool ID.
- `ApplyReport` carries: backup path, post-write fingerprint, whether a rollback occurred, structured per-key change list.
- `OwnedFile` carries: path, format, owned-key allowlist (`[]KeyPath`).
- `KeyPath` is a typed string with helpers for nested addressing (`env.ANTHROPIC_API_KEY`, `model_providers.openai.base_url`).
- Types live in `internal/writepath/types.go`; doc comments cite the PRD FR they encode.

**PRD/architecture refs.** PRD FR-5; Architecture §3, §4.

**Test plan.**
- Happy: construct and round-trip every type through Go test; verify diff helpers correctly compute added / removed / changed keys.
- Edge: `KeyPath` rejects empty / `.` / `..` / contains-`/`; nested-path resolver returns sentinel for missing-parent.

**No fallback writes.** N/A — pure types.

**Complexity.** S.

**Dependencies.** E1-S1.
