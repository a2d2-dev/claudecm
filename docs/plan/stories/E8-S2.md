# E8-S2 — CI workflow + ≥ 80% coverage gate on `internal/writepath`, `internal/commit`, `internal/resolver`

**User story.**
As a claudecm maintainer, I want CI to enforce the v1 coverage gates so silent regressions in the locked invariants are impossible.

**Acceptance criteria.**
- GitHub Actions workflow runs `gofmt`, `goimports`, `golangci-lint`, `go vet`, and `go test ./... -coverprofile=...` on every PR.
- Coverage report parses per-package totals; PR fails if `internal/writepath` < 80%, `internal/commit` < 80%, or `internal/resolver` < 80%.
- `go.mod` Go version is honored (pin per architecture/tech-stack).
- CI also runs the round-trip smoke (E8-S3) and the concurrent-edit / two-phase simulations (E8-S4, E8-S5).
- Coverage trend is recorded as a PR comment for visibility (optional, non-blocking).

**PRD/architecture refs.** PRD NFR-T1, SM-3; Architecture §11; coding-standards "Testing Mandates".

**Test plan.**
- Happy: representative PR with adequate coverage → CI green.
- Edge: drop a `writepath` test → CI fails the coverage gate.
