# E3-S5 — `claudecode`: `Apply` (calls `writepath.Apply`)

**User story.**
As a claudecm developer, I want `claudecode.Apply` to hand its single-file `WritePlan` to `internal/writepath.Apply` so that every Claude Code write goes through the FR-5 invariant and the adapter never opens the owned file for write itself.

**Acceptance criteria.**
- Given a `WritePlan` for `~/.claude/settings.json`, when `claudecode.Apply(plan)` is called, then it calls `writepath.Apply(plan)` and returns its `ApplyReport` plus structured warnings (NFR-S7).
- The adapter never calls `os.WriteFile`, `os.Rename`, or `os.Create` on `~/.claude/settings.json`. CI lint rule guards this.
- On `writepath.Apply` error, the adapter returns the error wrapped with `claudecode` context (`%w`).
- `Apply` is invoked by the commit package (E7) for the Claude Code file phase, never directly by `cmd/`.

**PRD/architecture refs.** PRD FR-5, FR-16; Architecture §3, §4; coding-standards rules 1, 13.

**Test plan.**
- Happy: end-to-end write of a profile via synthetic test → settings.json updated, backup created.
- Edge: writepath returns `ErrConcurrentEdit` → adapter surfaces it wrapped; lint check fails if any direct `os.Rename` is added.

**No fallback writes.** Adapter delegates; never has a write path of its own. FR-5.

**Complexity.** S.

**Dependencies.** E2-S2 + E2-S3 + E2-S4, E3-S4.
