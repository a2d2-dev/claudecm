# E3-S6 — `claudecode`: `Project` → `EffectiveView`

**User story.**
As a Claude Code user, I want `claudecm current` and `claudecm explain` to show what the tool will actually see, so I can sanity-check before a sensitive coding session.

**Acceptance criteria.**
- Given an active profile + the on-disk `~/.claude/settings.json`, when `claudecode.Project(profile)` runs, then it returns an `EffectiveView` listing each effective field with `WinningLayer`, `Source`, and `ShadowedLayers`.
- Default redaction (NFR-S8): `api_key` / `auth_token` rendered as `sk-***last4`.
- External drift: if the file's current SHA256 differs from `State.LastAppliedPerTool[claude_code].SHA256`, then `EffectiveView.ExternalDriftDetected = true` with the file path.
- `Project` does not write; it is read-only.

**PRD/architecture refs.** PRD FR-6, FR-7, NFR-S8, NFR-E1; Architecture §6, §6.2.

**Test plan.**
- Happy: profile sets `BASE_URL` + `MODEL`, on-disk file empty → `EffectiveView` reports overlay/core layers winning; redaction applies.
- Edge: env var `ANTHROPIC_API_KEY` set → EnvOverride wins, on-disk file marked as shadowed; external drift simulated by mutating the file → reported true.

**No fallback writes.** N/A — read-only.

**Complexity.** M.

**Dependencies.** E3-S4, E5-S1 (types).
