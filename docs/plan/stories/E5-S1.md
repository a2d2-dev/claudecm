# E5-S1 — Resolver types: `EffectiveView`, `EffectiveField`, `ShadowEntry`, `Layer`

**User story.**
As a claudecm developer, I want the resolver's value types declared in `internal/resolver/types.go` so adapters and the `current`/`explain` commands can compile against a stable contract before the chain logic lands.

**Acceptance criteria.**
- `Layer` is an enum: `BuiltInDefault`, `ProfileCore`, `ProfileOverlay`, `OnDiskToolConfig`, `EnvOverride`.

- `EffectiveView` carries `Tool ToolID`, `Fields []EffectiveField`, `ExternalDriftDetected bool`, `DriftFiles []string`.
- `ShadowEntry` carries `Layer`, `Value`, `Source`.
- Doc comments cite PRD FR-7 and architecture §6.
