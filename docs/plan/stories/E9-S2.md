# E9-S2 — Quickstart for SM-5 ≤ 3 min

**User story.**
As a new claudecm user with a working Claude Code or Codex install, I want a quickstart that takes me from `go install` to my first successful `switch` in under 3 minutes.

**Acceptance criteria.**
- A short quickstart page (or top-of-README section) walks: install → `claudecm import claude-code` (or `import codex`) → `claudecm switch <name>` → `claudecm current` → `claudecm explain`.
- Each step has a single copy-pasteable command and a one-line expected output.
- Timed against SM-5 on a warm machine without maintainer assistance.
- Includes a "what went wrong?" panel pointing at `explain`, `restore --list`, and the audit log.
- No mention of MCP, GUI, AES, Gemini, or any out-of-scope feature.

**PRD/architecture refs.** PRD SM-5, FR-1, FR-3, FR-4, FR-5, FR-8; Architecture §0.

**Test plan.**
- Happy: dry-run the quickstart manually + script it as an integration test (E8-S3 helps).
- Edge: missing tool install → quickstart instructs install of the tool first; SM-5 timing measured in CI smoke is < 3 min.

**No fallback writes.** N/A — docs.

**Complexity.** S.

**Dependencies.** E6-S4, E6-S2.
