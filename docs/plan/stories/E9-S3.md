# E9-S3 — `explain` + `switch --dry-run` asciinema demo

**User story.**
As a prospective claudecm user reading the README, I want a 60-second asciinema cast showing `explain` resolving an effective config and `switch --dry-run` printing the pre-apply diff, so I trust the tool before installing it.

**Acceptance criteria.**
- Two asciinema casts (or one combined cast) are recorded and embedded in the README.
- The cast uses a sandboxed `--home /tmp/claudecm-demo` so no real credentials are exposed.
- Default redaction (`sk-***last4`) is visible in the cast.
- The cast is regeneratable from a script committed alongside the asciinema file under `docs/demo/`.

**PRD/architecture refs.** PRD FR-4, FR-7, NFR-S8; Architecture §7.

**Test plan.**
- Happy: cast renders in GitHub README; redaction visible.
- Edge: any cast revealing a plaintext key triggers the CI grep guard (E8-S6) for `sk-` patterns.
