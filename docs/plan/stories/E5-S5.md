# E5-S5 — `cmd/explain` (winning + shadowed layers, redaction, `--reveal`, JSON output)

**User story.**
As a claudecm user, I want `claudecm explain` to show the full resolution chain — the winning layer plus every shadowed layer with its source — for both Claude Code and Codex, so I can debug billing, routing, and model surprises before they cost me.

**Acceptance criteria.**
- Given an active profile, when `explain` runs, then it prints per-tool sections; each effective field shows `WinningLayer`, `Source`, and shadowed layers in precedence order.
- Default redaction applies to `api_key` / `auth_token`; `--reveal` opts in and emits a stderr notice (NFR-S8).
- `--output json` emits machine-readable form for shell pipelines.
- External drift is surfaced prominently when present (E5-S4).
- `explain` ships and is acceptance-tested **before** `cmd/switch` (PRD §13 ordering note 5).
- 100% correctness on the explain fixture matrix (SM-4).

**PRD/architecture refs.** PRD FR-7, NFR-S8, NFR-E1, SM-4; Architecture §6, §7.

**Test plan.**
- Happy: profile + on-disk file + no env vars → chain printed; JSON output validates.
- Edge: `ANTHROPIC_API_KEY` env set → EnvOverride wins, on-disk file marked shadowed; `--reveal` shows plaintext + stderr notice.

**No fallback writes.** Read-only. NFR-S1.

**Complexity.** M.
