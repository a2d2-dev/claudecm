# E5-S4 — External-drift detection via `State.LastAppliedPerTool[tool].SHA256`

**User story.**
As a Claude Code or Codex user, I want `current` and `explain` to warn me when an owned tool file has been edited outside claudecm since the last successful `switch`, so I can decide whether to re-apply or re-import.

**Acceptance criteria.**
- After a successful `switch`/`restore`, `State.LastAppliedPerTool[tool]` records the file path + SHA256 + applied timestamp.
- `Project` re-hashes each owned file at read time; mismatch → `EffectiveView.ExternalDriftDetected = true` + `DriftFiles = [...]`.
- `current` prints a one-line warning when drift is detected; `explain` prints a prominent block listing affected files.
- Drift is **informational only** — no automatic re-apply, no automatic re-import. The user decides.
- When there's no prior applied state, drift is **not** reported (it would be noise).

**PRD/architecture refs.** PRD FR-6, FR-7; Architecture §2.2, §6.2.

**Test plan.**
- Happy: switch + immediate `current` → no drift.
- Edge: switch, then externally edit `~/.claude/settings.json` → `current` warns, `explain` lists the file; no prior state → no drift report.

**No fallback writes.** Drift detection never writes anything. Read-only.

**Complexity.** M.

**Dependencies.** E1-S7, E5-S2.
