# E8-S4 — Concurrent-edit simulation test per owned file (NFR-C2)

**User story.**
As a claudecm maintainer, I want a CI test per owned file (`auth.json`, `config.toml`, `settings.json`) that simulates an external edit between FR-5 step 2 and step 7 to assert the abort-with-backup behavior.

**Acceptance criteria.**
- A test for each of the three owned files mutates the target between writepath's read and rename phases (via a hook or controlled goroutine).
- The test asserts: `Apply` returns `ErrConcurrentEdit`; the backup created at step 6 is on disk; exit code (when surfaced through CLI) is 2.
- Tests are deterministic — no flaky sleeps; they use synchronization primitives.

**PRD/architecture refs.** PRD NFR-C2, NFR-C3; Architecture §4 step 9; coding-standards "Testing Mandates".

**Test plan.**
- Happy: per-file simulation → assertions hold.
- Edge: first-write race (file appears between step 2 and 7) → same `ErrConcurrentEdit`.

**No fallback writes.** NFR-C2 asserted: abort, keep backup. NFR-S1.

**Complexity.** M.
