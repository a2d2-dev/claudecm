# E9-S4 — Decision-log update (PRD §13)

**User story.**
As the claudecm maintainer team, I want any policy choice made during implementation that needs traceability to be recorded in PRD §13 so the decision log is the durable memory.

**Acceptance criteria.**
- After E1–E8 complete, the PRD §13 decision log is updated in one PR with any implementation-time policy choices (e.g., exact env-var precedence ties, JSON parser surprise, TOML round-trip quirks discovered).
- Each entry uses the same date + rationale format as the existing log entries.
- No silent re-litigation of ADR-0001-locked decisions; if implementation forces a change, that's a new ADR, not a §13 edit.
- The mirror at `_bmad-output/.../prd.md` is updated in the same commit (PRD mirror rule).

**PRD/architecture refs.** PRD §13 ("Decision Log"); ADR-0001; PRD mirror notice.

**Test plan.**
- Happy: log entries match the existing format.
- Edge: a proposed entry that conflicts with ADR-0001 → rejected; opens a new ADR instead.

**No fallback writes.** Docs only.

**Complexity.** S.

**Dependencies.** E1–E8 substantially complete.
