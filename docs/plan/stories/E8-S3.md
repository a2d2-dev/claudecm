# E8-S3 — CI round-trip smoke: `import → switch → export → explain`

**User story.**
As a claudecm maintainer, I want CI to run the full round-trip — import an on-disk fixture, switch to it, export, then explain — and assert byte-identity on owned-key scope, so SM-2 and SM-4 are mechanically enforced.

**Acceptance criteria.**
- A CI integration test exists at `internal/e2e/` (or similar) that runs against an isolated `--home /tmp/...`.
- For each tool fixture (canonical_minimal, canonical_maximal): import → save profile → switch → re-read tool file → assert byte-identical to fixture on owned-key scope; then `export` and `explain` to assert effective values are consistent.
- The smoke runs in < 5 s wall clock (covers SM-1 < 2 s for switch).
- The smoke fails if any owned-key fingerprint drifts.

**PRD/architecture refs.** PRD SM-1, SM-2, SM-4, NFR-T1; Architecture §11.

**Test plan.**
- Happy: minimal + maximal round-trip → green.
- Edge: corrupt a fixture to drift an owned key → smoke red with the drifted key in the error.
