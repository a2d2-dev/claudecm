# E8-S5 — Two-phase rollback simulation (Codex commits → Claude Code reparse fails → Codex restored)

**User story.**
As a claudecm maintainer, I want a CI test that simulates a partial-commit failure across the two tools and asserts the FR-16 rollback restores Codex byte-identically.

**Acceptance criteria.**
- A test stages plans for `auth.json`, `config.toml`, and `settings.json` and injects a reparse failure on `settings.json` after the Codex files commit successfully.
- The test asserts: both Codex files are rolled back byte-identical to their pre-commit content; `PartialFailure` reports Codex files as `rolled-back` and `settings.json` as `committed` but then rolled-back if applicable, with backup paths.
- Exit code propagation: 2 on clean rollback, 3 if injected rollback rename also fails.
- Deterministic; no flaky sleeps.

**PRD/architecture refs.** PRD FR-16, NFR-T1; Architecture §5; coding-standards "Testing Mandates".

**Test plan.**
- Happy (simulated failure): rollback restores Codex byte-identical → assertions hold.
- Edge: inject rollback-rename failure → exit 3, both errors surfaced in `PartialFailure`.

**No fallback writes.** Rollback is rename-only. FR-16.

**Complexity.** M.

**Dependencies.** E7-S4, E7-S5.
