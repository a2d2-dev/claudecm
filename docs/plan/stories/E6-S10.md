# E6-S10 — `cmd/list`

**User story.**
As a claudecm user, I want `claudecm list` to show all profiles with the active one marked so I can pick the next switch target without reading filesystem state by hand.

**Acceptance criteria.**
- Given a populated `~/.claudecm/profiles/` directory, when `list` runs, then each row shows `name`, `provider`, `model`, and an active marker (`*` or equivalent) against the profile referenced by `state.yaml.active`.
- Given default invocation (no `--reveal`), when `list` runs, then every `api_key` value is redacted as `<first4>...<last4>` (or `***` when the key length is < 8 characters); the raw token never appears on stdout or in JSON output.
- Given `--reveal`, when `list` runs, then the full `api_key` is rendered and a stderr notice is emitted per NFR-S8.
- Given `--output json`, when `list` runs, then stdout is a JSON array of profile objects (`name`, `provider`, `model`, `active`, `api_key`) honoring the same redaction policy as the text view; exit code 0 on success.
- Given an empty `~/.claudecm/profiles/` directory, when `list` runs, then it prints `no profiles` (text) or `[]` (JSON) and exits 0.
- Given any profile file that fails `internal/config` parse (corrupt YAML, missing `schema_version`, unknown version), when `list` runs, then it refuses with a non-zero exit, names the offending file, and does **not** fall back to a partial list (NFR-S1).

**PRD/architecture refs.** PRD FR-3, NFR-S8; Architecture §6 (redaction), §7 (CLI surface).

**Test plan.**
- Happy: three profiles on disk with one marked active in `state.yaml` → text output shows all three with the active marker on the right row; JSON output is a 3-element array with `active: true` on the right entry; default redaction applied to every `api_key`.
- Edge — empty store: `profiles/` directory exists but is empty → text prints `no profiles`, JSON prints `[]`, exit 0.
- Edge — short key: profile with `api_key` length < 8 (e.g., `sk-12`) → redacted as `***` (no first4/last4 leak).
- Edge — corrupt profile: one profile file has `schema_version` missing → command refuses with non-zero exit, error names the file, no partial list emitted, no rewrite.

**No fallback writes.** Read-only command. Corrupt profile → **refuse**, never auto-migrate, never partial-render. NFR-S1 / NFR-M1.
