# E6-S9 — `cmd/completion` + `cmd/version` + global flags (`--reveal`, `--home`, `--lock-timeout`, `--retention`)

**User story.**
As a claudecm user, I want shell completion that tab-completes profile names, a `version` command that reports semver + commit + build date, and global flags that consistently affect every command.

**Acceptance criteria.**
- `claudecm completion [bash|zsh|fish|powershell]` emits a working completion script; profile names tab-complete for `switch`, `delete`, `edit`, `rename`, `export`, `restore`.
- `claudecm version` reports `version`, `commit`, `build_date`; populated by `-ldflags` at build time. `pkg/version` exports the variables.
- `--reveal` global flag flips default redaction off in `list` / `current` / `explain` and emits a stderr notice (NFR-S8).
- `--home <path>` overrides `$HOME`; validated by `internal/storage/paths.go` (NFR-S3).
- `--lock-timeout <duration>` overrides the 5s flock timeout (NFR-C1).
- `--retention <int>` overrides backup retention per invocation (NFR-R1).
- Global flags are wired in `cmd/root.go` and consistent across all commands.

**PRD/architecture refs.** PRD FR-10, FR-11, NFR-S3, NFR-S8, NFR-C1, NFR-R1; Architecture §7.

**Test plan.**
- Happy: completion script source-loads in bash; `version` prints expected fields; each global flag is honored by `switch`.
- Edge: `--home /nonexistent` → refused; `--lock-timeout 0` → invalid; `--retention 0` → invalid.
