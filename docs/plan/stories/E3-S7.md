## Implementation Notes (post-review followup)

The fixture matrix landed in two passes; the review followup pass
added the missing edge cases the initial land skipped:

  - `edge/symlink-out-of-home/` — read-side containment check
    asserting `errors.Is(err, ErrOutsideHome)`.
  - `edge/missing/` — no settings.json seeded, asserts
    `errors.Is(err, ErrNoConfig)`.
  - `edge/bom/` — UTF-8 BOM prefix, asserts
    `errors.Is(err, ErrParseFailed)`.
  - `edge/comments/` — `//` line comment before the JSON root,
    asserts `errors.Is(err, ErrParseFailed)`.

Each error-only case is marked with an `error-only.txt` file
containing the expected error identifier. The fixture harness
routes those cases through an error assertion path and skips the
Import/Plan/Apply/Project golden compares.

The following AC-listed edge directories remain deferred to a
future story: `partial/`, `crlf/`, `mixed_indent/`, `separate_fs/`.
Each needs additional adapter-level design (whitespace-preserving
merge, CRLF policy, mixed-tabspace normalization, cross-fs rename
fallback) before a golden is meaningful.

Two related followups landed as story stubs:

  - `docs/plan/stories/E2-FOLLOWUP-flatten-nil.md` — Flatten(nil)
    surfaces the `""` key on an empty current; blocks Plan+Apply
    goldens for `edge/empty/` and `edge/whitespace-only/`.
  - `docs/plan/stories/E2-FOLLOWUP-symlink-in-home-apply.md` —
    writepath Apply should follow in-HOME symlinks; blocks the
    Plan+Apply golden for `edge/symlink-in-home/`.

Two tripwires guard against future silent gaps:

  - `TestFixtureGoldensAreValidJSON` — every `expected/*.json`
    must round-trip through `json.Unmarshal`.
  - `compareCount` inside `runFixtureCase` — a non-error-only case
    that skipped every golden compare (all goldens missing) fails
    the test rather than passing silently.
